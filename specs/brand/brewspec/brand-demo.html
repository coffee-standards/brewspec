Positioning

"BrewSpec is an open data format for describing coffee brews — a shared schema that lets tools exchange, store, and analyse brew data without building their own from scratch."

BrewSpec is an open, versioned data format for describing a coffee brew — method, dose, water, grind, duration, extraction, result. It's free, Apache 2.0, and built on JSON Schema. If you're building a tool that handles brew data, this is a shared format your data can travel in.

Structured data. Validated schema.

A BrewSpec file is a YAML or JSON document describing a single brew record. The schema defines which fields are required, which are optional, and what values are valid — using standard JSON Schema. Any conforming validator can check a brew record against it.

This is what BrewSpec does: give tool builders a shared format with a defined contract, so data produced by one tool can be read by any other without custom parsing.

dose_g number Coffee dose in grams. Required. Must be > 0.
water_g number Water weight in grams. Required. Must be > 0.
tds number Total dissolved solids as percentage. Optional.
extraction_yield_pct number Extraction yield percentage. Optional. 0–100.
method enum Brew method. Required. One of the defined values.
